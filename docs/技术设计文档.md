# 「轻养随行」V1.0 MVP 技术设计文档

## 一、项目概述

### 1.1 产品背景
- **产品名称**：「轻养随行」
- **产品形态**：微信小程序（V1.0 MVP版）
- **核心目标**：解决上班族办公 8 小时核心亚健康问题，提供碎片化、低门槛、隐秘化的基础健康干预方案
- **产品定位**：上班族碎片化亚健康基础管家

### 1.2 目标用户
| 人群分类 | 年龄段 | 核心痛点 |
|---------|-------|---------|
| 年轻白领 | 22-28岁 | 颈椎酸痛、用眼疲劳、久坐不动 |
| 加班中坚 | 29-38岁 | 腰椎劳损、久坐不活动、忘记喝水 |
| 资深职场人 | 39-42岁 | 慢性肩颈痛、久坐不适、饮水量不足 |

### 1.3 现有项目技术栈
| 类别 | 技术选型 |
|-----|---------|
| UI组件库 | WeUI + Vant Weapp |
| 样式语言 | 推荐 Less |
| 开发框架 | 微信小程序原生框架 |
| 代码规范 | ESLint + Prettier |
| 分包机制 | 主包(3个TabBar) + 8个独立分包 |

## 三、功能模块详细设计

### 3.1 模块1：智能定时提醒系统

#### 3.1.1 功能描述
根据用户设置的上班时间，推送微信服务消息提醒

#### 3.1.2 数据结构设计

```javascript
// 提醒设置数据结构
interface ReminderSettings {
  // 上班时间（小时，0-23）
  workStartHour: number;      // 默认 9
  workEndHour: number;        // 默认 18

  // 久坐提醒
  sedentaryEnabled: boolean;  // 默认 true
  sedentaryInterval: number;  // 默认 45（分钟），范围 30-120

  // 护眼提醒
  eyeCareEnabled: boolean;    // 默认 true
  eyeCareInterval: number;    // 默认 60（分钟），范围 30-120

  // 喝水提醒
  waterEnabled: boolean;      // 默认 true
  waterInterval: number;      // 默认 90（分钟），范围 30-120

  // 免打扰设置
  doNotDisturbEnabled: boolean; // 默认 false（后续迭代）
}

// 存储键名
const REMINDER_SETTINGS_KEY = 'reminder_settings';
```

#### 3.1.3 交互逻辑
1. **首次进入**：弹出上班时间设置引导
2. **定时触发**：使用 `wx.setInterval` + 本地通知记录实现
3. **提醒方式**：本地缓存提醒时间，下次进入时计算是否需要提醒

#### 3.1.4 页面原型（提醒设置页 ID:04）

### 3.2 模块2：工位微运动库

#### 3.2.1 功能描述
满足「坐姿、小幅度、零器械、隐秘」的 1-5 分钟动作，共17个核心动作

#### 3.2.2 动作数据结构

```javascript
// 运动动作数据结构
interface ExerciseAction {
  id: string;              // 动作唯一标识
  name: string;            // 动作名称
  category: 'neck' | 'waist' | 'posture' | 'wrist';  // 分类
  duration: number;        // 时长（分钟）
  difficulty: 'easy' | 'medium' | 'hard';  // 难度
  icon: string;            // 图标名称
  isFavorite: boolean;     // 是否收藏
  steps: ExerciseStep[];   // 动作步骤
}

interface ExerciseStep {
  stepNumber: number;      // 步骤编号
  imageUrl: string;        // 步骤示意图
  description: string;     // 步骤描述
}
```

#### 3.2.3 动作分类详情

| 分类 | 分类名称 | 动作数量 | 动作列表 |
|-----|---------|---------|---------|
| neck | 颈椎舒缓 | 6个 | 点头仰头、左顾右盼、颈部旋转、耸肩环绕、侧颈拉伸、收颌回归 |
| waist | 腰椎放松 | 4个 | 坐姿扭转、猫式伸展、骨盆倾斜、坐姿前弯 |
| posture | 体态矫正 | 5个 | 坐姿挺胸、肩胛骨内收、背部扩展、胸廓扩张、脊柱延伸 |
| wrist | 手腕放松 | 2个 | 手腕旋转、手腕拉伸 |

#### 3.2.4 打卡记录结构

```javascript
// 打卡记录
```

### 3.4 模块4：喝水记录

#### 3.4.1 功能描述
定时提醒喝水，记录饮水量

#### 3.4.2 数据结构

```javascript
// 喝水记录
interface WaterRecord {
  date: string;           // 日期 YYYY-MM-DD
  totalIntake: number;    // 今日总饮水量（ml）
  goalIntake: number;     // 目标饮水量（ml），默认 1500
  unitSize: number;       // 每次饮水量（ml），默认 200
  records: WaterIntakeRecord[];  // 饮水明细
}

interface WaterIntakeRecord {
  time: number;          // 时间戳
  amount: number;         // 饮水量（ml）
}

// 存储键名
const WATER_RECORD_KEY = 'water_record';        // 喝水记录
const WATER_SETTINGS_KEY = 'water_settings';   // 喝水设置
```

#### 3.4.3 交互逻辑
1. 每次点击「已喝水」自动累加默认值（200ml）
2. 首次达标（≥1500ml）弹出鼓励文案
3. 支持手动调整每次饮水量

## 十一、风险与注意事项

### 11.1 技术风险
1. **微信服务消息限制**：模板消息需要用户主动触发，需提前设计好消息模板
2. **本地定时不准**：微信小程序不支持真实的后台定时任务，需结合用户启动时计算
3. **存储容量限制**：小程序本地存储上限为10MB，需考虑数据清理策略

### 11.2 交互风险
1. **打卡体验**：PRD强调「一键打卡」，避免过多填写步骤
2. **进度反馈**：需在显眼位置展示今日进度，增强用户成就感
3. **首次引导**：上班时间设置需简洁，避免用户流失

### 11.3 后续迭代预留
1. 免打扰时段功能（PRD暂不实现）
2. 数据同步与云存储
3. 社交分享能力
